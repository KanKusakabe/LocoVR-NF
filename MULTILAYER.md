# 多層NF — 同じ空間に「環境の層」と「個人・行動の層」を積む

**親**: [LocoVR-NF](README.md)(実住宅の導線尤度で家具配置を採点・再設計)。その2つのフローを**横並びから縦積み**に変える拡張トラック。
**問い**: 配置・デザインを決めるNFと、実際に人がどう動くかのNFを**別々に保ちつつ、同じ座標空間で合成する**と、単層NFには出せない出力が得られるか。
作成 2026-07-13。親 `.venv`(torch2.12 / zuko1.6 / MPS)、`PYTHONPATH=. uv run --no-project python -m locovrnf.{filter,coarse,compare}`。設計の元文書 = [`../future-plan/multilayer-nf-experiment-plan.md`](../future-plan/multilayer-nf-experiment-plan.md)。

---

## 出発点:材料はもう手元にある

LocoVR-NF は同じ座標空間の上に、駆動要因の違う2つのNFを**別々に**学習済み。多層化とは、これを縦に積んで合成する一手を足すこと。

| レイヤー | 実体(既存) | NFが表すもの | 変数空間 |
|---|---|---|---|
| **環境層 E**(配置・デザイン) | `AffordanceFlow` = `p(居場所 \| 占有)` | レイアウトが決める foot-traffic 場 | 位置 |
| **個人・行動層 U**(実利用) | `StepFlow` = `p(次ステップ \| 占有, goal, 速度)` | 実際の歩き方・経路の引かれ方 | 変位 |

⚠ 非対称:環境層は「位置」の密度、行動層は「変位」の密度。素朴に掛け算できない。この整合の取り方が2トラックの分かれ目。

## 一言でわかる結果(フェーズ制・ゲート制、負の結果も正直に)

### トラックB — 積 = 各ステップのパーティクルフィルタ(横のモジュール性)
2つの専門家を**別々に学習**し、推論時のみ各ステップで合成:エージェント(StepFlow)が候補を**提案**し、環境専門家(AffordanceFlow)が各着地点を `weight ∝ exp(進捗/τ + β·logp_E)` で**再重み付け**。β=環境の結合ツマミ。**手製の clearance 場は一切使わない**(学習した密度がその役を確率的に担えるかを見る)。エージェント・環境とも**未知レイアウト3を除く3配置で学習**。

| Ph | 何を測ったか(holdout=未知レイアウト3, k=32本) | 結果 | 判定 |
|---|---|---|---|
| **B0 到達** | フィルタが実際にゴールへ到達するか(成功率↑) | β=2で成功率 **0.80**(>0.5) | ✅ |
| **B1 中核** | β掃引で家具重なり率↓、minADE非悪化 | 重なり **0.370→0.353→0.322→0.285**(β=0/1/2/4)単調減、minADE 0.541→0.497(悪化なし) | ✅ |
| **B2 モジュール性★** | 未知レイアウトで、後付けの環境層が重なりを下げるか | 重なり **0.370(環境なし)→0.285(環境あり)= 23%減**、環境は**再学習せず推論時に合成のみ** | ✅ |

到達成功率は β=0/1/2/4 で 0.80/0.84/0.80/0.74 と保たれる(高βでやや低下=環境を重視するほど遠回りになる、正直な傾向)。

### トラック粗→細 — 大域intent flow → 局所dynamics flow(縦の大域/局所分離)
スケールの違う2フローを縦積み:**粗**フロー `p(次ウェイポイントへのオフセット \| 占有, goalベアリング)` が大域計画を、**細**フロー(既存StepFlow)がウェイポイント間を歩く。ウェイポイント=実軌跡の転回点。**4レイアウトの leave-one-layout-out**(H3)。

| Ph | 何を測ったか(4分割LOO平均) | 結果 | 判定 |
|---|---|---|---|
| **H0 表現** | ウェイポイント骨格が直線より導線を捉えるか(ADE↓[m]) | 骨格 **0.047m** ≪ 直線 0.89m | ✅ |
| **H1 生成** | 粗→細の生成が直線基線を超えるか(minADE↓[m]) | 粗→細 **0.682m** < 直線 0.89m | ✅ |
| **H1補足(正直)** | 明示clearanceを持つ手製コントローラを超えるか | 粗→細 0.682m は手製 **0.547m に及ばず**(手製は硬い障害物判定を持つ) | ⚠ 未達・明記 |
| **H2 大域識別★★** | 粗フロー尤度が「実迂回 vs 家具を突っ切る直線」を区別するか | **粗尤度で実経路が上=82%**(全4分割 0.68–0.88)。一方**per-step尤度では実が上=16%のみ** | ✅ C3の失敗を反転 |

---

## 何が「単層では出せない」のか(2トラックの核心)

- **B2(横のモジュール性)**: 環境密度は**占有条件付き**なので、**一度学習すれば未知レイアウトの占有を入れるだけ**で新しい家具を避ける。エージェントを触らずに推論時オーバーレイで動線を引き直せる=「レイヤーを重ねる」を実現。
- **H2(縦の大域識別)**: `A1/A2/C3` で3回躓いた「**局所per-step尤度は大域導線を測れない**」(C3では直線の方がper-step尤度が高かった)を、**粗フローの経路尤度が構造で回復**。per-step は実経路を16%しか上位にしないが、粗フローは82%上位にする。**全4レイアウトで一貫して反転**。

X1 の含意:両トラックは同等の経路分布を生むが、**Bは推論時の層差替、Hは大域経路識別**という別々の能力を足す=**相補的**。粗フローの下層(細)に積Bの環境専門家を差すと縦×横の三層になる(X2=次段)。

---

## 正直な制約(既存プロジェクトの地雷を先に踏まない)

- **局所尤度 ≠ 大域導線**(`A1/A2/C3`)。B・H とも経路品質の主張は**生成分布の minADE と家具重なり率**のみに置き、**per-step 尤度ランキングでは主張しない**。H2 はむしろ「per-step が失敗する」ことを対照として使う。
- **位置 vs 変位の非対称**。素朴な密度積は不可 → Bは各ステップの重点リサンプリング(フィルタ)で回避。
- **識別性**。B の環境専門家・エージェントはともに**未知レイアウトを除外**して学習(leak-free)。
- **粗→細は手製clearanceコントローラを超えない**(H1補足)。Track H の固有貢献は H2(大域識別)であり、手製プランナの出し抜きではない——ここは隠さず boolean で報告。

## モデル(既存骨格の再利用)

- `locovrnf/filter.py` — トラックB。`train_affordance`(環境専門家をtrain-layoutのみで学習)+ `rollout_filter`(提案=StepFlow、選択=前進候補の中を `exp(β·logp_E)` で。前進候補が無い粒子は純進捗にフォールバックし到達性を保つ)。
- `locovrnf/coarse.py` — トラック粗→細。`waypoints`(転回点抽出)+ `train_coarse`(粗フロー、StepFlow骨格をウェイポイント尺度で)+ `coarse_plans_batch`/`fine_batch`(k粒子バッチのロールアウト)+ `coarse_logp_route`(H2の経路尤度)。
- `locovrnf/compare.py` — X1。保存済み JSON を集約し比較表・図を生成。

## 出力(`results/`)

- `b_filter.{json,png}` — β掃引(重なり・minADE・成功率)/ β=0 vs β=best のルート束(未知レイアウト)
- `h_coarse.{json,png}` — 粗計画+細充填 / H0/H1 の route error 棒 / **H2 の反転**(per-step は直線を、粗は実経路を上位に)
- `x_compare.{json,png}` — 6手法の minADE・家具重なり比較 + 2つの固有能力(B2/H2)

```bash
uv run --no-project python -m locovrnf.coarse --holdout 3 --loo    # H0/H1/H2/H3 (~7min)
uv run --no-project python -m locovrnf.compare                     # X1

```
